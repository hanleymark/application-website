MARK HANLEY

VISION IMPAIRED DEVELOPER

Introduction

My name is Mark Hanley and I am a wannabe developer who believes passionately that the web must be accessible to all. In particular, I want to use my personal understanding of vision loss and its challenges, to build web sites that are truly accessible to vision-impaired people.

I'm going to tell you why I strongly believe that I possess the skills, experience and motivation to become a key member of the next team of F&C future developers!

My story (abridged)

First of all...

// I've always loved coding.

From silly animations and games back in the distant days of the home-micro revolution when I was a kid, to coding up Towers of Hanoi solutions in Excel to See If It Can Be Done.

I've always harboured the ambition to make coding my full-time job. However, having been a teacher for over twenty years, I've had to fit in the fun little programming projects around my main job. Consequently, my tinkering with Python, C#, Excel macros and more... has never attained the depth and saturation of knowledge required to make the leap into professional software development.

// Then something happened...

On Friday 13th March 2020, while I was just about to enjoy my morning coffee before heading off to work, the entire right half of my visual field in both eyes went black. Disappeared. In a matter of seconds, without warning or pain, I permanently lost half my vision.

What I didn't know was that, while I had been sleepily preparing my coffee, a tiny blood clot had formed in my body and made its way up to the left occipital lobe in my brain and wreaked havoc in my visual cortex. The impact on my vision can be seen in these visual field perimetry test results...

Cut to now, 2022. Thanks to meds and a minor operation to patch a hole in my heart, what happened to me on that day is very unlikely to happen again. It has taken time and a lot of learning, but I have fully come to terms with my disability. Teaching young children however, with a serious vision impairment, has undeniably been tough. Meeting the complex needs of a class of thirty children when you can't see half of them at any one time is a constant challenge that can take its toll. It's time for a change of direction. And what better way than finally to realise my ambition to code as my profession.

I may have lost my vision, but it has opened my eyes. Life is short and things can change big-style in the blink of an eye. Make time to chase your dreams.

// And my dream is to be a web developer...

...front-end, back-end or even full-stack. And I fully believe that the Founders & Coders Skills Bootcamp is the best way for me finally to achieve that dream.

Portfolio

I want to show this journey, warts and all. Therefore I have not gone back to the earlier projects to improve them using knowledge, skills and techniques that I have learned more recently.

Tribute page project

Gallery project

COMMENT BOX PROJECT

Comment box project

This was my first attempt at serious form input. The project involved taking user input, validating it and then displaying validated content on the page. It also involved dynamically updating the user on the current status of some input fields.

Key learning:

  • Adding elements to the page dynamically
  • Form validation
  • Custom form validation error reporting

Next steps:

  • Make it prettier!
  • Add ARIA tags to form to improve accessibility
MOVIE DATA PROJECT

Movie data project

This felt like a much bigger project which provided a much greater challenge. The brief left lots of room for interpretation which enabled me to choose where to focus my learning. I decided to attempt much more ambitious CSS in order to develop a clear page style.

Key learning:

  • More complex CSS styling
  • Using tables to layout dynamic information
  • More pseudo classes
  • Modal windows in CSS
  • More form validation
  • Dynamically show/hide content for smaller viewports
  • Add interactivity with edit/add record functionality
  • Use of Google Fonts API

Next steps:

  • Add ARIA tags to improve accessibility
  • Refine modal window layout on smaller devices
JAVASCRIPT GAME PROJECT

Javascript game project

This was my favourite project! An almost pure JavaScript game with very little HTML/CSS.

I decided to have a go at creating a clone of the fiendishly difficult but addictive 2048 game. The game involves using the arrow keys to slide blocks, each with a number on, across a grid. When two blocks of the same value are slid into each other, they combine and their value doubles. Each time the blocks move, a randomly generated block with a 2 or 4 value is placed in a free grid space. The game is over when no more new blocks can be placed and no slides are possible. I loved the constant problem solving that this project brought and the challenge of developing the gameplay algorithms.

I would like to further develop this in the future to use CSS to draw the board rather than an HTML canvas. This would enable me to focus on animating the tiles as they move.

Key learning:

  • LOTS of JavaScript!
  • Use of JavaScript classes for encapsulation and to improve code readability
  • WCAG compliant contrasting colour combinations
  • HTML canvas drawing
  • Capturing and handling keyboard input

Next steps:

  • Animate the block slides smoothly using CSS delta T animations
  • Add game controls for touch devices (game is not playable without a keyboard)
APPLICATION SITE PROJECT

Application site project

The last project! I wanted to develop a pleasing style and layout which showed many of the reasons why I want to join the wonderful F&C Bootcamp programme. A key area to improve would be the structure of the CSS as the single CSS file has grown huge and somewhat difficult to follow. I would like to investigate how I can use CSS frameworks like SASS and separate CSS files to achieve this.

Key learning:

  • Using anchors for navigation
  • Fixed navigation links at the top of the page
  • Play CSS animations when the user scrolls them into the viewport
  • Further develop CSS styling

Next steps:

  • Structure CSS mmuch ore effectively
  • Ensure consistent styling on all browsers
  • Continue to add projects from my own learning
  • Become a Founder & Coders software developer apprentice!
